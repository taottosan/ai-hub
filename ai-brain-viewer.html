❓ Phase 1: Question

The AI receives the user's question and begins processing. This is where the Action Router determines what kind of request this is.

POST /api/v1 {"action": "context.build", "params": {"user_id": "user_1", "query": "What's the user's preferred theme and how do I set it?"}}

Key decisions at this stage:

  • 📌 Classify intent (question, command, follow-up)
  • 📌 Extract entities (user_id, theme, preference)
  • 📌 Route to appropriate handler (context.build)

📋 What Each Phase Contributes

❓ Question

Parses intent, extracts entities, classifies the request type. Routes to the correct handler.

🧠 Memory

Retrieves episodic memories — past conversations, user preferences, session history. Ranked by recency × importance.

📚 Knowledge

Queries semantic knowledge — documentation, APIs, procedures. Retrieved from Obsidian, Skills, Archive providers.

⚡ Skill

Matches the task to a reusable procedure — a step-by-step guide the AI has been trained to follow for common operations.

🔧 Tool

Executes the chosen tool call — API request, database query, file write — and returns the result to the AI Brain.

✨ Answer

Assembles the final response using the enriched context. Includes the explainability trace with a unique trace_id.
